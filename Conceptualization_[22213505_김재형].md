# 1. Conceptualization

**Project Title:** 날씨 맞춤형 외출복 코디 추천시스템

---

## 1. Business purpose

**project background**
우리 생활속에 필요한 것을 말하라고 하면 대부분 의식주를 말한다.그중 의에 해당하는 옷을 우리는 고민하느라 시간을 많이 할애한다.특히 환절기나 기온변화가 심한 날에는 일기예보의 온도 숫자만 보고 옷차림을 결정했다가 춥거나 더워서 낭패를 보는 경우가 많다.바쁜 아침 출근 시간에 옷장을 뒤적이며 날씨에 맞는 상의와 하의, 아우터를 조합하는 과정은 불필요한 시간 낭비와 결정 피로를 유발한다.
그래서 '내가 가진 옷들을 종류와 두께별로 텍스트 파일에 분류해 두고, 오늘 외출할 시간의 기온만 입력하면 적절한 코디 조합을 바로 출력해 주는 가벼운 프로그램이 있다면 아침 준비 시간을 크게 단축시킬수 있지 않을까?'라는 생각에 본 프로젝트를 기획하게 되었다.

**project goal**
-사용자가 기온만 입력하면, 날씨에 가장 잘 맞는 옷 조합(상의, 하의, 겉옷)을 즉시 3가지 추천해준다.
-귀찮게 옷 사진을 찍어 올릴 필요 없이, 메모장에 텍스트로만 옷 이름을 적어두면 프로그램이 알아서 인식하고 관리한다.
-매일 아침 "뭐 입지?" 고민하느라 낭비되는 시간을 획기적으로 줄여주어 외출 준비의 스트레스를 없앤다.

## 3. Use case list

| ID | Use Case Name (기능명) | Actor (행위자) | Description (상세 설명) |
| **UC-01** | **시스템 시작 및 데이터 로드** | System | 프로그램 실행 시 `clothing_list.txt` 파일에서 기존 옷장 데이터를 읽어와 메모리(객체)에 로드한다. |
| **UC-02** | **메인 메뉴 출력 및 명령어 대기** | System | 터미널 창에 사용자가 선택할 수 있는 기능 목록(1~5번 등)을 텍스트로 깔끔하게 출력하고 입력을 대기한다. |
| **UC-03** | **현재 기온 입력 및 코디 추천** | User | 외출할 시간의 기온(예: 15도)을 숫자로 입력하면, 시스템이 기온에 맞는 상의, 하의, 아우터 조합 3가지를 추천해준다. |
| **UC-04** | **추천 코디 재검색 (Re-roll)** | User | 추천된 코디가 마음에 들지 않을 경우, 특정 키를 입력하여 동일한 온도 조건에서 다른 조합을 다시 추천받는다. |
| **UC-05** | **새로운 옷 정보 등록** | User | 새로 구매한 옷의 종류(Top/Bottom/Outer), 이름, 두께 정보를 텍스트로 입력하여 시스템에 등록한다. |
| **UC-06** | **옷장 전체 목록 조회** | User | 현재 등록된 옷장 안의 모든 옷 리스트를 번호표와 함께 화면에 스크롤 형식으로 출력하여 확인한다. |
| **UC-07** | **카테고리별 옷 필터링 조회** | User | 전체 목록 대신 '상의만 보기', '아우터만 보기' 등 특정 클래스(카테고리)의 옷들만 필터링하여 조회한다. |
| **UC-08** | **기존 옷 정보 수정** | User | 조회된 옷의 고유 번호를 입력하여, 옷의 이름이나 두께 등 잘못 입력된 세부 정보를 수정한다. |
| **UC-09** | **안 입는 옷 정보 삭제** | User | 낡아서 버렸거나 더 이상 입지 않는 옷의 번호를 입력하여 메모리 상의 옷장 리스트에서 영구 삭제한다. |
| **UC-10** | **시스템 종료 및 데이터 저장** | User, System | 종료 명령어를 입력하면, 시스템이 변경된 최신 옷장 리스트를 텍스트 파일에 안전하게 덮어쓰기(Save)한 후 프로그램을 종료한다. |
