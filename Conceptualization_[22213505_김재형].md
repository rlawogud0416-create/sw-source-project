# 1. Conceptualization

**Project Title:** 날씨 맞춤형 외출복 코디 추천시스템

---

## 1. Business purpose

**project background**
                       <img width="189" height="267" alt="image" src="https://github.com/user-attachments/assets/c1e17695-9922-4bdc-a611-608c96b6f0ba" />



우리 생활속에 필요한 것을 말하라고 하면 대부분 의식주를 말한다.그중 의에 해당하는 옷을 우리는 고민하느라 시간을 많이 할애한다.특히 환절기나 기온변화가 심한 날에는 일기예보의 온도 숫자만 보고 옷차림을 결정했다가 춥거나 더워서 낭패를 보는 경우가 많다.바쁜 아침 출근 시간에 옷장을 뒤적이며 날씨에 맞는 상의와 하의, 아우터를 조합하는 과정은 불필요한 시간 낭비와 결정 피로를 유발한다.
그래서 '내가 가진 옷들을 종류와 두께별로 텍스트 파일에 분류해 두고, 오늘 외출할 시간의 기온만 입력하면 적절한 코디 조합을 바로 출력해 주는 가벼운 프로그램이 있다면 아침 준비 시간을 크게 단축시킬수 있지 않을까?'라는 생각에 본 프로젝트를 기획하게 되었다.

**project goal**
-사용자가 기온만 입력하면, 날씨에 가장 잘 맞는 옷 조합(상의, 하의, 겉옷)을 즉시 3가지 추천해준다.
-귀찮게 옷 사진을 찍어 올릴 필요 없이, 메모장에 텍스트로만 옷 이름을 적어두면 프로그램이 알아서 인식하고 관리한다.
-매일 아침 "뭐 입지?" 고민하느라 낭비되는 시간을 획기적으로 줄여주어 외출 준비의 스트레스를 없앤다.

## 3. Use case list

**1)시스템 시작 및 데이터 로드**<br>
Actor:System<br>
Description:프로그램 실행 시 `clothing_list.txt` 파일에서 기존 옷장 데이터를 읽어와 메모리에 로드한다.<br>
**2)메인 메뉴 출력 및 명령어 대기**<br>
Actor:System<br>
Description:터미널 창에서 사용자가 선택할 수 있는 기능 목록을 텍스트로 깔끔하게 출력하고 입력을 대기한다.<br>
**3)현재 기온 입력 및 코디 추천**<br>
Actor:User<br>
Description:외출할 시간의 기온을 숫자로 입력하면, 시스템이 기온에 맞는 상의, 하의, 아우터 조합 3가지를 추천해준다.<br>
**4)추천 코디 재검색**<br>
Actor:User<br>
Description:추천된 코디가 마음에 들지 않을 경우, 특정 키를 입력하여 동일한 온도 조건에서 다른 조합을 다시 추천받는다.<br>
**5)새로운 옷 정보 등록**<br>
Actor:User<br>
Description:새로 구매한 옷의 종류(Top/Bottom/Outer), 이름, 두께 정보를 텍스트로 입력하여 시스템에 등록한다.<br>
**6)옷장 전체 목록 조회**<br>
Actor:User<br>
Description:현재 등록된 옷장 안의 모든 옷 리스트를 번호표와 함께 화면에 스크롤 형식으로 출력하여 확인한다.<br>
**7)카테고리별 옷 필터링 조회**<br>
Actor:User<br>
Description:전체 목록 대신 '상의', '아우터' 등 특정 클래스(카테고리)의 옷들만 필터링하여 조회한다.<br>
**8)기존 옷 정보 수정**<br>
Actor:User<br>
Description:조회된 옷의 고유 번호를 입력하여, 옷의 이름이나 두께 등 잘못 입력된 세부 정보를 수정한다.<br>
**9)안 입는 옷 정보 삭제**<br>
Actor:User<br>
Description:낡아서 버렸거나 더 이상 입지 않는 옷의 번호를 입력하여 메모리 상의 옷장 리스트에서 영구 삭제한다.<br>
**10)시스템 종료 및 데이터 저장**<br>
Actor:User, System<br>
Description:종료 명령어를 입력하면, 시스템이 변경된 최신 옷장 리스트를 텍스트 파일에 덮어쓰기한 후 프로그램을 종료한다.<br>

## 4. Concept of Operation

**1)시스템 초기화 및 데이터 로드**<br>
-Purpose:프로그램 실행 시 기존 옷장 데이터를 메모리에 적재한다.<br>
-Approach:프로그램 시작 시 로컬 경로의 파일(wardrobe.txt)을 탐색하여 기존에 저장된 의류 데이터(카테고리, 이름, 두께 등)를 파싱한 뒤, C++ 객체로 생성하여 메모리의 리스트에 추가한다.<br>
-Dynamics:사용자가 처음 프로그램을 실행할 경우<br>
-Goals:사용자가 이전에 저장해 둔 옷장 상태를 유실 없이 그대로 불러와 사용할 수 있게 한다.<br>

**2)메인 메뉴 출력 및 명령어 대기**<br>
-Purpose:사용자에게 시스템의 기능을 안내하고 입력을 대기한다.<br>
-Approach:CLI(터미널) 환경에 텍스트로 구성된 메뉴를 출력하고, 사용자의 키보드 입력을 받는다. 지정되지않은 명령어가 입력되면 예외 처리하여 다시 입력받는다.<br>
-Dynamics:데이터 로드가 완료된 후나, 특정 기능 수행이 완료되어 기본 화면으로 돌아온 경우<br>
-Goals:사용자가 시스템의 기능들을 쉽게 파악하고 원하는 명령을 안정적으로 실행할 수 있도록 돕는다.<br>

**3)신규 의류 등록**<br>
-Purpose:사용자가 새로 구매하거나 등록할 옷을 시스템에 추가할 수 있게 한다.<br>
-Approach:사용자가 안내에 따라 카테고리(상의/하의/겉옷), 이름, 두께 레벨(1~5단계)을 입력하면, 시스템은 새로운 의류 객체를 생성하여 메모리 상의 옷장 리스트에 추가한다.<br>
-Dynamics:사용자가 메뉴에서 새로운 옷 추가 기능을 선택한 경우<br>
-Goals:사용자의 실제 옷장 변화를 시스템 내부에 정확하게 반영할 수 있게 한다.<br>

**4)기존 의류 삭제**<br>
-Purpose:사용자가 더 이상 입지 않는 옷을 시스템에서 제거한다.<br>
-Approach:현재 등록된 옷장 목록을 출력하고, 삭제할 옷의 고유 번호를 입력받아 시스템 메모리 리스트에서 해당 객체를 완전히 삭제한다.<br>
-Dynamics:사용자가 메뉴에서 기존 옷 삭제 기능을 선택한 경우<br>
-Goals:불필요한 데이터를 삭제하여 코디 추천 시 안 입는 옷이 나오는 것을 방지하고 관리 효율을 높인다.<br>

**5)옷장 데이터 조회 및 필터링**<br>
-Purpose:시스템에 등록된 옷의 전체 또는 특정 카테고리 목록 확인할 수 있게 한다.<br>
-Approach:메모리에 있는 객체 리스트를 순회하며 조건(전체 조회 또는 '상의만 보기' 등)에 맞는 객체의 정보를 텍스트 표 형태로 화면에 깔끔하게 출력한다.<br>
-Dynamics:사용자가 내 옷장 전체 보기 또는 특정 카테고리 필터링 메뉴를 선택한 경우<br>
-Goals:사용자가 현재 자신이 보유한 옷의 상태(종류 및 두께 레벨)를 직관적으로 파악할 수 있게 한다.<br>

**6)기온 맞춤형 코디 추천**<br>
-Purpose:사용자가 입력한 기온에 가장 적합한 외출복 조합을 추천한다.<br>
-Approach:사용자가 오늘 예상 기온(예: 15)을 입력하면, 시스템은 내부 기준표를 통해 적정 두께 레벨을 산출한다. 해당 두께 조건에 맞는 상/하/겉옷 객체만 필터링한 뒤, 랜덤으로 1세트를 조합하여 화면에 출력한다.<br>
-Dynamics:사용자가 외출 전 메뉴에서 기온을 입력하고 코디 추천을 요청한 경우<br>
-Goals:날씨와 옷의 두께를 논리적으로 매칭하여 최적의 코디를 빠르게 제안함으로써 사용자의 결정 피로를 없앤다.<br>

**7)시스템 종료 및 데이터 영구 저장**<br>
-Purpose:변경된 옷장 데이터를 영구적으로 보존하고 시스템을 종료한다<br>.
-Approach:종료 명령이 입력되면, 현재 메모리 상에 존재하는 최신화된 의류 객체 리스트의 정보들을 문자열로 변환하여 wardrobe.txt 파일에 덮어쓰기로 저장한 뒤 시스템 자원을 반납한다.<br>
-Dynamics:사용자가 프로그램 사용을 모두 마치고 종료를 요청한 경우.<br>
-Goals:파일 입출력을 통해 데이터 손실 없이 최신 상태를 하드디스크에 안전하게 기록하고 프로그램을 닫는다.<br>

## 5. Problem statement

본 시스템은 현실에 존재하는 사용자의 의류를 데이터화하여 관리하고, 외부 환경 변수에 맞춰 최적의 결과물을 산출해야 한다. 이를 위해 시스템이 설계 및 구현 단계에서 해결해야 할 핵심 과제들은 다음과 같다.<br>

**1)의류 속성의 정량화 및 객체화**<br>
시스템이 다루어야 할 대상인 '옷'은 종류가 매우 다양할 뿐만 아니라 '두께감'이나 '계절감'처럼 사람마다 주관적으로 느끼는 정성적인 속성을 지닌다. 시스템이 이를 논리적으로 연산하고 특정 기온 데이터와 비교하기 위해서는, 이러한 주관적 요소를 1~5단계와 같은 객관적 수치로 변환하는 기준이 필요하다. 더불어 각기 다른 특징을 가진 상의, 하의, 겉옷을 C++의 클래스로 온전히 추상화하여 시스템 메모리상에 구현해야 한다.<br>

**2) 환경 변수 기반의 추천 알고리즘 설계**<br>
시스템은 사용자가 입력한 오늘의 '기온'이라는 환경 변수를 정확하게 해석하여, 전체 옷장 데이터 중 해당 기온에 부합하는 두께 레벨을 가진 객체들만 선별해 내는 필터링 과정을 거쳐야 한다. 단순히 옷을 나열하는 것을 넘어, 필터링된 객체들 사이에서 상/하의 및 겉옷이 누락 없이 하나의 완전한 세트로 조합되도록 알고리즘을 설계해야 하며, 이 모든 과정이 지연 없이 즉각적으로 처리되어야 한다.<br>

**3) 데이터 계층 구조화 및 영속성(Persistence) 확보**<br>
수십 벌의 옷 데이터를 효율적으로 탐색하고 관리하기 위해, 다형성을 활용한 객체지향적 계층 분류 체계가 확립되어야 한다. 또한, 무거운 외부 데이터베이스를 배제한 CLI 응용프로그램의 특성상 프로그램 종료 시 메모리상의 데이터가 휘발되는 문제가 발생한다. 따라서 로컬 텍스트 파일을 활용해 객체 데이터를 직렬화하고, 시스템이 켜고 꺼질 때마다 데이터를 안전하게 동기화하여 영구적으로 보존할 수 있는 구조가 요구된다.<br>

**4) 텍스트 기반 사용자 경험(UX) 최적화**<br>
시각적인 그래픽이 제공되지 않는 터미널 환경에서 운용되므로, 사용자가 텍스트 명령어만으로도 시스템의 모든 기능을 직관적으로 제어할 수 있어야 한다. 특히, 잘못된 명령어 입력이나 문자/숫자 혼용 입력 등으로 인해 프로그램이 강제 종료되는 현상을 방지하기 위한 견고한 예외 처리 로직이 필수적이며, 시스템의 응답 결과를 깔끔하고 정돈된 텍스트 포맷으로 출력하여 사용자의 피로도를 낮춰야 한다.<br>

## 6.Glossary

-CLI (Command Line Interface):마우스나 그래픽 창(GUI) 없이, 터미널 화면에서 키보드의 텍스트 입력만으로 컴퓨터와 소통하는 인터페이스 방식. 본 시스템의 기본 구동 환경이다.<br>
-다형성 (Polymorphism):하나의 부모 타입(Clothing) 포인터나 참조자로 다양한 자식 타입(Top, Bottom 등)의 객체들을 일관되게 제어할 수 있는 객체지향의 핵심 성질. 본 시스템에서 옷장 리스트 내의 다양한 옷들을 동일한 명령(예: 정보 출력)으로 처리할 때 사용된다.<br>
-직렬화:시스템 구동 중 메모리에 동적 할당되어 있던 의류 객체 데이터들을, 프로그램 종료 시 파일 시스템에 영구 보존하기 위해 텍스트 문자열 형태로 변환하여 기록하는 과정이다.<br>
-두께 레벨:옷의 두께와 계절감을 컴퓨터가 계산하기 쉽도록 1단계(여름용)부터 5단계(겨울용)까지 정수로 정량화한 자체 기준이다.<br>
