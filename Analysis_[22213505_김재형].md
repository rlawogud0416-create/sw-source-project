## 1. Introduction

본 문서는 'WC(Weather Coordinator) - 날씨 맞춤형 외출복 코디 추천 프로그램' 프로젝트의 Analysis 단계 산출물이다. 앞선 Conceptualization 단계에서 정의한 시스템의 목적과 기능을 바탕으로, 시스템이 가져야 할 기능을 Use case 관점에서 분석하고, 시스템을 구성하는 도메인 클래스를 식별하며, 사용자가 실제로 마주하게 될 CLI(Command Line Interface) 화면의 초안을 설계하는 것을 목표로 한다.

### 1.1 Project features

**(1) 유용성(Usefulness)**

매일 아침 옷을 고르느라 낭비되는 시간과 결정 피로를 획기적으로 줄여준다. 사용자는 본인의 옷장을 한 번 등록해 두기만 하면, 이후로는 외출할 시각의 기온만 입력해도 즉시 적절한 코디 조합을 추천받을 수 있다. 환절기처럼 옷차림이 까다로운 시기에 특히 유용하다.

**(2) 중요성(Significance)**

기존의 코디 추천 서비스 대부분은 옷의 사진을 일일이 촬영·업로드해야 하거나, 외부 쇼핑몰의 신상품을 추천하는 마케팅 도구에 가깝다. 본 시스템은 무거운 GUI나 외부 라이브러리 없이도 '내가 이미 가지고 있는 옷'을 텍스트 기반으로 가볍게 등록하여, 옷의 두께 레벨이라는 객관적 수치를 기준으로 기온에 맞는 조합을 산출한다는 점에서 차별성을 가진다. 또한 C++의 객체지향 특성(상속, 다형성, 캡슐화)을 적극 활용하여 의류 데이터를 체계적으로 관리한다.

**(3) 확장성(Expandability)**

본 시스템은 '두께 레벨'이라는 단일 속성을 중심으로 설계되어 있으나, 추후 색상·스타일·소재·세탁 가능 여부 등 다양한 속성을 추가하여 매칭 알고리즘을 고도화할 수 있다. 또한 현재는 사용자가 직접 입력한 기온만을 사용하지만, 향후 외부 기상 데이터 연동이나 GUI 버전으로의 이식도 가능하도록 클래스 구조가 분리되어 있다. 데이터 저장 방식 또한 텍스트 파일에서 데이터베이스로 자연스럽게 교체될 수 있는 구조로 설계된다.

| 역할 | 시스템이 다루는 모든 옷의 공통 속성과 동작을 정의하는 **추상 클래스**이다. |
| 주요 속성 | `id`(고유 번호), `name`(옷 이름), `thicknessLevel`(두께 레벨, 1~5) |
| 주요 메서드 | `getCategory()`(순수 가상 함수, 자식 클래스에서 구현), `display()`(가상 함수, 콘솔 출력), `toFileString()`(가상 함수, 파일 저장용 문자열 반환) |
| 비고 | Top, Bottom, Outer 클래스가 이 클래스를 상속받아 다형성을 구현한다. `vector<Clothing*>`로 다양한 자식 객체를 일관되게 관리할 수 있다. |

#### (2) Top (상의)

| 항목 | 내용 |
| :--- | :--- |
| 역할 | Clothing을 상속받아 '상의' 카테고리를 표현하는 클래스이다. |
| 비고 | `getCategory()` 메서드는 "Top"을 반환한다. |

#### (3) Bottom (하의)

| 항목 | 내용 |
| :--- | :--- |
| 역할 | Clothing을 상속받아 '하의' 카테고리를 표현하는 클래스이다. |
| 비고 | `getCategory()` 메서드는 "Bottom"을 반환한다. |

#### (4) Outer (아우터)

| 항목 | 내용 |
| :--- | :--- |
| 역할 | Clothing을 상속받아 '아우터' 카테고리를 표현하는 클래스이다. |
| 비고 | `getCategory()` 메서드는 "Outer"를 반환한다. |

#### (5) Wardrobe (옷장)

| 항목 | 내용 |
| :--- | :--- |
| 역할 | 사용자가 보유한 모든 의류 객체를 관리하는 컨테이너 클래스이다. |
| 주요 속성 | `clothingList`(`vector<Clothing*>`), `nextId`(다음에 부여할 고유 번호) |
| 주요 메서드 | `add()`, `remove()`, `update()`, `findById()`, `filterByCategory()`, `filterByThickness()`, `displayAll()` |
| 관계 | Clothing과 1:N 합성(Composition) 관계를 가진다. 소멸자에서 모든 자식 객체의 메모리를 해제할 책임이 있다. |

#### (6) OutfitRecommender (코디 추천기)

| 항목 | 내용 |
| :--- | :--- |
| 역할 | 입력된 기온을 분석하여 옷장에서 적절한 의류를 선별·조합하는 핵심 로직 클래스이다. |
| 주요 메서드 | `tempToThickness(temp)`(기온→두께 레벨 매핑, static 메서드), `recommend(temp, wardrobe)`(코디 1세트 반환) |
| 관계 | Wardrobe를 사용하여(uses) 의류 데이터에 접근한다. |

#### (7) Outfit (코디)

| 항목 | 내용 |
| :--- | :--- |
| 역할 | 추천된 한 세트의 코디(상의 + 하의 + 아우터)를 표현하는 데이터 객체이다. |
| 주요 속성 | `top`, `bottom`, `outer` (각 자식 클래스의 포인터) |
| 주요 메서드 | `display()`(코디 정보를 콘솔에 텍스트 박스 형태로 출력), `isValid()`(세 가지 카테고리 모두 채워졌는지 확인) |

#### (8) DataStorage (데이터 저장소)

| 항목 | 내용 |
| :--- | :--- |
| 역할 | Wardrobe 객체의 영속성(persistence)을 담당하는 클래스이다. |
| 주요 메서드 | `save(wardrobe, path)`(`<fstream>`을 이용해 텍스트 파일로 직렬화 저장), `load(wardrobe, path)`(파일에서 읽어 의류 객체로 역직렬화하여 옷장에 적재) |
| 비고 | static 메서드로 구현하여 인스턴스 생성 없이 호출한다. 카테고리 문자열을 보고 적절한 자식 객체(`new Top`, `new Bottom`, `new Outer`)를 생성하는 팩토리 역할도 겸한다. |

#### (9) CLIView (콘솔 화면 제어)

| 항목 | 내용 |
| :--- | :--- |
| 역할 | 콘솔 입출력을 담당하는 화면 제어 클래스이다. 비즈니스 로직과 화면 출력을 분리한다. |
| 주요 메서드 | `showMainMenu()`(메인 메뉴 출력 후 사용자 선택 반환), `showWardrobe()`(옷장 목록 출력), `showRecommend()`(추천 결과 출력), `getInput()`(범용 입력 함수), `showError()`(오류 메시지 출력) |
| 비고 | 추후 GUI로 확장하더라도 이 클래스만 교체하면 되도록 설계한다. (MVC 패턴의 View에 해당) |

### 3.3 기온-두께 매핑 규칙

## 4. User Interface prototype

본 시스템은 CLI(Command Line Interface) 환경에서 동작하며, 모든 입출력은 키보드와 콘솔 화면을 통해 이루어진다. 아래는 각 주요 화면의 텍스트 목업이다.

### 4.1 메인 메뉴 화면

```
============================================================
         WC - Weather Coordinator (v1.0)
            날씨 맞춤형 외출복 코디 추천
============================================================
  현재 등록된 옷: 6벌 (상의 2 / 하의 2 / 아우터 2)
------------------------------------------------------------
  [ 1 ] 옷 추가하기
  [ 2 ] 옷장 보기
  [ 3 ] 옷 정보 수정하기
  [ 4 ] 옷 삭제하기
  [ 5 ] 코디 추천받기
  [ 0 ] 프로그램 종료
============================================================
  메뉴를 선택하세요 >> _
```

[그림 4-1] 메인 메뉴 화면

**사용 설명**

(1) 사용자는 화면 하단의 프롬프트(`>>`)에 메뉴 번호(0~5)를 입력하고 Enter를 누른다.

(2) 잘못된 번호나 문자를 입력하면 오류 메시지가 출력되고 다시 입력받는다.

(3) 화면 상단에는 현재 옷장 상태가 요약되어 표시된다.

### 4.2 옷 추가 화면

```
============================================================
                     [ 옷 추가 ]
============================================================
  카테고리를 선택하세요:
    [ 1 ] 상의 (Top)
    [ 2 ] 하의 (Bottom)
    [ 3 ] 아우터 (Outer)
  선택 >> 1

  옷 이름을 입력하세요:
  >> 회색 긴팔 티셔츠

  두께 레벨을 입력하세요 (1: 가장 얇음 ~ 5: 가장 두꺼움):
  >> 3

------------------------------------------------------------
  ✓ 등록 완료!
    [#7] 상의 / 회색 긴팔 티셔츠 / Lv.3
------------------------------------------------------------
  Enter 키를 눌러 메인 메뉴로 돌아가세요...
```

[그림 4-2] 옷 추가 화면

**사용 설명**

(1) 메인 메뉴에서 `1`을 선택하면 이 화면으로 진입한다.

(2) 카테고리 → 이름 → 두께 레벨 순서로 차례대로 입력받는다.

(3) 잘못된 입력값이 들어오면 해당 단계에서 다시 입력받는다.

(4) 등록이 완료되면 즉시 `wardrobe.txt`에 저장된다.


## 5. Glossary

| 용어 | 설명 |
| :--- | :--- |
| WC | 본 프로젝트의 이름(Weather Coordinator)이다. |
| CLI (Command Line Interface) | 마우스나 그래픽 창(GUI) 없이 터미널 화면에서 키보드의 텍스트 입력만으로 컴퓨터와 소통하는 인터페이스 방식이다. 본 시스템의 기본 구동 환경이다. |
| 사용자 (User) | 프로그램을 실행하여 옷장 등록 및 코디 추천 기능을 이용하는 사람이다. |
| 카테고리 | 옷의 종류를 의미하며, 본 시스템에서는 상의(Top), 하의(Bottom), 아우터(Outer)로 구분한다. |
| 두께 레벨 | 옷의 두께와 계절감을 정량화한 자체 기준으로, 1단계(여름용)부터 5단계(한겨울용)까지 5단계로 구성된다. |
| 코디 (Outfit) | 상의, 하의, 아우터가 각각 1개씩 포함된 한 벌의 옷 조합을 의미한다. |
| Wardrobe | 사용자가 보유한 모든 의류 객체를 담는 컨테이너 클래스이다. |
| 다형성 (Polymorphism) | 부모 클래스(Clothing) 타입의 포인터로 자식 클래스(Top, Bottom, Outer) 객체를 일관되게 다루는 객체지향의 성질이다. |
| 직렬화 (Serialization) | 메모리상의 객체를 파일에 저장하기 위해 문자열(`id,category,name,thickness` 형식)로 변환하는 과정이다. |
| 역직렬화 (Deserialization) | 파일에 저장된 문자열을 다시 메모리상의 객체로 복원하는 과정이다. |
| 추상 클래스 (Abstract class) | 순수 가상 함수를 1개 이상 가지는, 직접 인스턴스화할 수 없는 클래스이다. 본 시스템의 Clothing 클래스가 이에 해당한다. |
| 동적 할당 (Dynamic allocation) | `new` 연산자로 힙(heap) 메모리에 객체를 생성하는 방식이다. 사용 후 `delete`로 반드시 해제해야 한다. |
| 텍스트 파일 입출력 | C++ 표준 라이브러리 `<fstream>`의 `ifstream`/`ofstream`을 통해 텍스트 파일을 읽고 쓰는 기능이다. |
| Use case | 시스템이 사용자에게 제공해야 할 기능 단위를 행위자(Actor)와 시스템 간의 상호작용으로 표현한 것이다. |
